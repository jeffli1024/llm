What is BPE?

Byte Pair Encoding is a data-compression algorithm repurposed for text tokenization. Originally described by Philip Gage in 1994 for lossless compression, it was adapted for NLP by Sennrich et al. in 2016 and now underpins tokenizers in GPT-2, GPT-4, LLaMA, Mistral, and many others.

The core idea is simple: repeatedly merge the most frequent pair of adjacent symbols until you reach a target vocabulary size. This creates a vocabulary of subword units that balances character-level flexibility with word-level efficiency.

Why Subword Tokenization?

Character-level models handle any word but sequences are very long. Word-level models are efficient but suffer from out-of-vocabulary (OOV) tokens — unseen words become <UNK>. Subword methods give us the best of both worlds: a fixed vocabulary of moderate size (typically 32k–100k) where every word — no matter how obscure — can be expressed as a sequence of known subword tokens.

The BPE Training Algorithm

Given a corpus, first count word frequencies. Represent each word as a sequence of characters with a special end-of-word symbol ⬜ (or </w>). Then iterate:

// Corpus after pre-tokenization
5 × l o w ⬜
2 × l o w e r ⬜
6 × n e w e s t ⬜
3 × w i d e s t ⬜

Count all adjacent pairs. The pair e s appears 6+3=9 times — the most frequent. Merge it → es. Repeat.

Pair Frequency Counting

For each word with frequency f, every adjacent pair (a, b) in the word contributes f to that pair's count. The total pair count is:

count(a,b) = Σw ∈ corpus freq(w) × occurs(a,b,w)

Where occurs(a,b,w) is the number of times pair (a,b) appears consecutively in word w.

The Word-End Symbol

A special marker (often </w> or Ġ prefix in GPT-2) is appended/prepended to mark word boundaries. This prevents merges that would cross word boundaries — so l o w and e r from different words would never merge together. GPT-2 uses a space prefix Ġ so Ġlow represents "low" at the start of a word or after a space.

Encoding New Text (Inference)

At inference time, to tokenize a new string: split on whitespace (and apply any pre-tokenization regex). For each word, start with individual characters. Apply saved merge rules in order — earlier merges take priority. This greedy application of the learned rules is deterministic and fast.

// Encoding "lowest" with learned rules
Start: l o w e s t ⬜
Rule 1 (e s → es): l o w es t ⬜
Rule 2 (es t → est): l o w est ⬜
Rule 3 (est ⬜ → est⬜): l o w est⬜
Rule 4 (l o → lo): lo w est⬜ ← final tokens
Byte-Level BPE (GPT-2 style)

GPT-2 operates on raw UTF-8 bytes rather than Unicode characters. The base vocabulary is all 256 byte values. This guarantees every possible string encodes without any unknown tokens — even emoji, Chinese characters, or arbitrary binary data. The downside is slightly longer sequences for non-English text, since each Unicode code point may require 2–4 bytes.

Time Complexity

Naïve BPE training: for V merge operations on a corpus of N characters, a naïve scan is O(V·N). With priority queues and incremental updates it can be brought to roughly O(N log N). In practice, training GPT-2's tokenizer on WebText took only minutes.

BPE in Python — Annotated
1. Build the initial vocabulary from a corpus
def get_vocab(corpus): # Split each word into chars + end-of-word marker vocab = {} for word in corpus.split(): # "low" → ("l","o","w","") chars = tuple(list(word) + ["</w>"]) vocab[chars] = vocab.get(chars, 0) + 1 return vocab
2. Count all adjacent symbol pairs
def get_pairs(vocab): pairs = {} for word, freq in vocab.items(): symbols = list(word) for i in range(len(symbols) - 1): pair = (symbols[i], symbols[i+1]) pairs[pair] = pairs.get(pair, 0) + freq return pairs
3. Merge the best pair everywhere in the vocab
def merge_vocab(best_pair, vocab): new_vocab = {} bigram = " ".join(best_pair) # "e s" merged = "".join(best_pair) # "es" import re pattern = re.compile( r"(?<!\S)" + re.escape(bigram) + r"(?!\S)" ) for word in vocab: word_str = " ".join(word) new_word_str = pattern.sub(merged, word_str) new_word = tuple(new_word_str.split()) new_vocab[new_word] = vocab[word] return new_vocab
4. Main training loop
def train_bpe(corpus, num_merges): vocab = get_vocab(corpus) merge_rules = [] for i in range(num_merges): pairs = get_pairs(vocab) if not pairs: break # Find most frequent pair (ties broken by sort) best = max(pairs, key=lambda p: pairs[p]) vocab = merge_vocab(best, vocab) merge_rules.append(best) print(f"Merge {i+1}: {best[0]} + {best[1]} → {''.join(best)}") return vocab, merge_rules # Example corpus = "low low low low low lower lower newest newest newest newest newest newest widest widest widest" vocab, rules = train_bpe(corpus, num_merges=10)
5. Encoding new text
def encode(word, merge_rules): # Start with characters tokens = list(word) + ["</w>"] for (a, b) in merge_rules: i = 0 while i < len(tokens) - 1: if tokens[i] == a and tokens[i+1] == b: tokens[i:i+2] = [a + b] # merge else: i += 1 return tokens # "lower" → ["low", "er", "</w>"] (using learned rules) print(encode("lower", rules))
